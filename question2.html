Answer to Question 2

GIT is version control software that can be operated in two ways: via a graphical user interface (GUI), or through a command line interface (CLI).

Advantages of Command Line Interface (CLI)

The benefits of the command line in the life of a developer are clear, especially when it comes to using Git.

  • If you need to do something like SSH tunnel into remote servers and run operations, the CLI is the only interface that allows you to do it.
  • With a simple script you can schedule git commands automatically, letting you do things like update a remote server at exactly 2:00am every single day.
  • No matter what your development platform, you can use the command line to do what you want. Relatedly, if you look for help with Git online, you'll probably find directions for the command line.

Advantages of Graphical user Interface (GUI).

Graphical interfaces can perform many helpful functions for the active developer.

  • A GUI can save you many lines of code with a drag-and-drop or the push of a button
  • GUIs help take care of a lot of work that can be obscure. GUIs help present the user with advanced options that they might not know exist otherwise
  • The barrier-of-entry for GUIs is much lower than for the command line. Novice developers can become effective faster and can ultimately get more done.
  • GUIs are best able to graphically represent collaboration. Branches, changes to branches, teammate activity, and other such information is best conveyed visually.
